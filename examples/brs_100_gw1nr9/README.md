# Brisbane Silicon BRS-100-GW1NR9 Reference Design

This example is a Gowin reference design for the Brisbane Silicon
BRS-100-GW1NR9 board. It targets the `GW1NR-LV9QN88PC7/I6` device and
instantiates the fpgacapZero Gowin ELA wrapper through the `GW_JTAG`
primitive.

The design is deliberately small: it builds one ELA, drives the board LEDs
from simple local status signals, and captures internal counters, buttons, and
header I/O through a six-channel probe mux.

## Files

| File | Purpose |
| --- | --- |
| `brs_100_gw1nr9_top.v` | Top-level reference design |
| `brs_100_gw1nr9_loc.cst` | Gowin pin constraints |
| `brs_100_gw1nr9_timing.sdc` | Clock and false-path constraints |
| `build.py` | Preferred Gowin batch-build launcher |
| `build_brs_100_gw1nr9.tcl` | Gowin project/script used by `build.py` |
| `brs_100_gw1nr9.cfg` | OpenOCD config for the board JTAG adapter |

## Design Summary

- Target part: `GW1NR-LV9QN88PC7/I6`, device version `C`
- Input clock: 27 MHz board oscillator
- Sample clock: 60 MHz generated by `rPLL`
- JTAG helper clock: 18 MHz generated by `rPLL`
- ELA sample width: 8 bits
- ELA depth: 64 samples
- ELA channels: 6
- EIO: disabled in this example

The ELA probe bus is split into six 8-bit channels:

| Channel | Captured signal |
| --- | --- |
| 0 | 8-bit free-running counter |
| 1 | `{6'b0, button[1:0]}` |
| 2 | `pad_io[8:1]` |
| 3 | `pad_io[16:9]` |
| 4 | `pad_io[24:17]` |
| 5 | `pad_io[32:25]` |

## Board I/O

The LEDs are active-low at the pads and are driven from these internal status
signals:

| LED | Meaning |
| --- | --- |
| 0 | 1 Hz heartbeat |
| 1 | JTAG activity indicator |
| 2 | User button 0 |
| 3 | User button 1 |
| 4 | `pad_io[1]` |
| 5 | `pad_io[2]` |

The two user buttons are active-low on the board and are sampled into the
60 MHz domain before being exposed through ELA channel 1.

## Build

From the repository root:

```sh
python examples/brs_100_gw1nr9/build.py
```

The build writes Gowin project outputs under:

```text
examples/brs_100_gw1nr9/out/
```

The generated bitstream is:

```text
examples/brs_100_gw1nr9/out/fcapz_brs_100_gw1nr9.fs
```

If `gw_sh` is not on `PATH`, pass either the Gowin installation directory or
the `gw_sh` executable explicitly:

```sh
python examples/brs_100_gw1nr9/build.py --gowin /path/to/Gowin
python examples/brs_100_gw1nr9/build.py --gowin /path/to/gw_sh
```

## Program And Connect

Program `out/fcapz_brs_100_gw1nr9.fs` with Gowin Programmer or your usual
Gowin programming flow.

Then start OpenOCD from the repository root:

```sh
openocd -f examples/brs_100_gw1nr9/brs_100_gw1nr9.cfg
```

In another terminal, probe the ELA through the OpenOCD backend:

```sh
fcapz --backend openocd --host 127.0.0.1 --port 6666 \
  --tap GW1NR-9C.tap probe
```

The CLI auto-selects the Gowin IR table when the tap name starts with `GW`.
The checked-in OpenOCD config names the tap `GW1NR-9C.tap`.

## Capture Examples

Capture the free-running counter on channel 0:

```sh
fcapz --backend openocd --host 127.0.0.1 --port 6666 \
  --tap GW1NR-9C.tap \
  capture --channel 0 --pretrigger 8 --posttrigger 16 \
  --trigger-value 0 --trigger-mask 0xff \
  --out brs_counter.json --format json
```

Capture the button channel:

```sh
fcapz --backend openocd --host 127.0.0.1 --port 6666 \
  --tap GW1NR-9C.tap \
  capture --channel 1 --pretrigger 8 --posttrigger 16 \
  --trigger-value 0 --trigger-mask 0xff \
  --out brs_buttons.json --format json
```

For VCD output, change `--format json` to `--format vcd` and use a `.vcd`
output path.

## Notes

- This example uses Gowin register-path ELA readback, not Xilinx-style burst
  readback.
- OpenOCD must already be running; the fpgacapZero OpenOCD transport does not
  program the FPGA.
- The example does not include hardware regression tests yet.
